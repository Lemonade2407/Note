# 绪论
## 数据库系统概述
### 4个基本概念
1. 数据Data
2. 数据库DB
3. 数据库管理系统（数据管理软件）
4. 数据库系统（数据库+数据库管理系统+应用程序+数据库管理员）DBS
### 数据管理技术的产生和发展
1. 人工管理阶段
	- 数据不保存
	- 应用程序管理数据
	- 数据不共享
	- 数据不具有独立性
2. 文件系统阶段
	- 数据长期保存
	- 数据由专门软件管理
	- 数据共享性弱，冗余度高
	- 数据独立性弱
3. 数据库系统阶段
	- 整体数据结构化：主要特征与本质区别
	- 数据的共享性强、冗余度低且易于扩充
	- 数据的独立性强：显著优点
		- 物理独立性
		- 逻辑独立性
	- 数据由数据库管理系统统一管理和控制
		- 数据的安全性保护
		- 数据的完整性检查
		- 数据的并发控制
		- 数据库的恢复
## 数据模型
- 数据库系统的核心和基础
### 概念模型
- 用于信息世界的建模
#### 信息世界概念
- 实体entity
- 属性attribute
- 码key
- 实体类型entity type
- 实体集entity set
- 联系relationship
#### 实体-联系模型
### 数据模型三要素
- 数据结构：描述对象与对象的关系
- 数据操纵
- 完整性约束
### 层次模型
- 树状结构，一对多
#### 数据结构
- 有且只有一个节点没有双亲节点，这个节点称为根节点
- 根以外的节点只有一个双亲节点
#### 优缺点
- 优点
	- 数据结构简单清晰
	- 查询效率高
	- 良好的完整性约束支持
- 缺点
	- 现实中有些情况不适用
	- 程序编写复杂
	- 查询子女必须通过双亲节点
	- 结构严密，命令趋于程序化
### 网状模型
- 网状结构，多对多
#### 数据结构
- 允许一个以上的节点无双亲
- 一个节点可以有多于一个的双亲节点
