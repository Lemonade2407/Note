# 三层架构
- controll：控制层，接收前端请求，处理请求，响应数据
	- 接受前端请求：用@RequestParam
	```
	public Result delete(@RequestParam("id") Integer id){
	}
	若请求参数名与形参名相同，则可省略@RequestParam，即
	public Result delete(Integer id){
	}
	```
	- 接受json格式：用@RequestBody
	```
	public Result delete(@RequestBody Dept dept){
	}
	json数据键名与形参对象的属性的名相同
	```
	- 接收参数：参数用{...}标识，用@PathVariable获取
	```
	@GetMapping("/depts/{id}")
	public Result getInfo(@PathVariable("id")Integer deptId)
	若路径参数名与形参名相同，则可省略为
	public Result getInfo(@PathVariable Integer deptId)
	```
- service：业务逻辑层，处理逻辑相关
- dao：数据访问层，负责增删改查
# 分层解耦
- 控制反转(IOC)：对象的创建控制权转移给容器
	- 实现类加上注解`@Component`(基础注解)、`@RestController`(标注在控制层)、`@Service`(标注在业务层)、`@Repository`(标注在数据访问层)
- 依赖注入(DI)：容器为应用程序提供运行时所依赖的资源
	- 对象加上注解`@Autowired`(默认按类型装配)、`@Aotuwired+@Qualifier("实现类名称")`(指定名称)、`@Resource("实现类名称")`、实现类加上注解`@Primary`（优先注入）
- Bean对象：IOC容器中的对象
