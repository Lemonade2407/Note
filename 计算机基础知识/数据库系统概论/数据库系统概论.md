# 绪论
## 数据库系统概述
### 4个基本概念
1. 数据Data
2. 数据库DB
3. 数据库管理系统（数据管理软件）
4. 数据库系统（数据库+数据库管理系统+应用程序+数据库管理员）DBS
### 数据管理技术的产生和发展
1. 人工管理阶段
	- 数据不保存
	- 应用程序管理数据
	- 数据不共享
	- 数据不具有独立性
2. 文件系统阶段
	- 数据长期保存
	- 数据由专门软件管理
	- 数据共享性弱，冗余度高
	- 数据独立性弱
3. 数据库系统阶段
	- 整体数据结构化：主要特征与本质区别
	- 数据的共享性强、冗余度低且易于扩充
	- 数据的独立性强：显著优点
		- 物理独立性
		- 逻辑独立性
	- 数据由数据库管理系统统一管理和控制
		- 数据的安全性保护
		- 数据的完整性检查
		- 数据的并发控制
		- 数据库的恢复

### 层次模型
- 树状结构，一对多
#### 数据结构
- 有且只有一个节点没有双亲节点，这个节点称为根节点
- 根以外的节点只有一个双亲节点
#### 优缺点
- 优点
	- 数据结构简单清晰
	- 查询效率高
	- 良好的完整性约束支持
- 缺点
	- 现实中有些情况不适用
	- 程序编写复杂
	- 查询子女必须通过双亲节点
	- 结构严密，命令趋于程序化
### 网状模型
- 网状结构，多对多
#### 数据结构
- 允许一个以上的节点无双亲
- 一个节点可以有多于一个的双亲节点
#### 优缺点
- 优点
	- 更直接描述现实世界
	- 性能良好，存取效率高
- 缺点
	- 结构复杂
	- DDL和DCL复杂，不易使用
	- 编写程序负担重
### 关系模型
#### 数据结构
- 关系：通常对应一个二维表
- 元组：表中一行
- 属性：表中一列
- 码：用于确定唯一元组的一个或一组属性
- 域：属性的取值范围
- 分量：元组中的一个属性值
- 关系模式：描述关系，一般表示为：关系名（属性1，属性2，......，属性n）
####  优缺点
- 优点
	- 建立在严格的数学概念基础上
	- 概念单一
	- 存取路径对用户隐蔽
- 缺点
	- 查询效率低

## 数据库系统的三级模式结构
### 模式的概念
- 模式也称逻辑模式，是数据库中全体数据的逻辑结构和特征的描述，具体值称为实例
- 模式反映的是数据的结构及其联系，实例反映的是数据库某一时刻的状态
### 三级模式结构
- 模式、外模式、内模式
#### 模式
- 模式是所有用户的公共数据视图，一个数据库对应一个模式
#### 外模式
- 也称子模式/用户模式，是数据库用户能够看见和使用的局部数据的逻辑结构和特征的描述，是数据库用户的数据视图，一个数据库可以有多个外模式
#### 内模式
- 也称物理模式/存储模式，是对数据物理结构和存储方式的描述，是数据在数据库内部的组织方式，一个数据库只有一个内模式

# 关系模型
## 数据结构及形式化定义
### 关系
- 域($D_1,D_2,...,D_n$)：一组具有相同数据类型的值的集合
	- 基数($M=\Pi^n_{i=1}m_i$)：域的不同取值个数
- 笛卡尔积($D_1\times D_2\times\dots\times D_n$)：域上的集合运算
	- 元组($d_1,d_2,...,d_n$)：笛卡尔积的每个元素
	- 分量($d_i$)：元组中的每一个值
- 关系($R(D_1,D_2,...,D_n)$)：笛卡尔积的有限子集
	- 基本关系
		- 列是同质的：分量来自同一个域、有相同数据类型
		- 不同列可出自同一个域
		- 列无序
		- 码不可重复
		- 行无序
		- 分量不可再分
	- 查询结果
	- 视图
### 关系模式
- 关系模式（$R(U,D,DOM,F)$)：关系的描述
	- R为关系名，U为属性名集合，D为域，DOM为属性向域的映像（说明分别出自哪个域）集合，F为属性间数据依赖关系的集合
	- DOM直接说明为属性的类型、长度
- 候选码：可以唯一标识一个元组的属性（组），诸属性被称为主属性，其余为非主（码）属性
- 主码：从候选码中选定一个，在名称下加下划线表示
- 全码：所有属性都是候选码
### 关系数据库
- 关系数据库系统：支持关系模型的数据库系统
- 关系数据库模式：对关系数据库的描述，所有关系模式的集合
- 关系数据库：关系模式对应的关系的集合

## 关系的完整性
- 关系的值应满足的条件
### 实体完整性（必须满足）
- 主属性不能取空
### 参照完整性（必须满足）
- R的一组属性F与S的主码K相对应
	- F是R的外码、R为参照关系、S为被参照关系
- F取值为空或等于S的某个主码值
### 用户定义的完整性
## 关系代数
- 用关系运算表达查询
### 传统的集合运算
- 并、差、交、笛卡尔积
### 专门的关系运算
- 选择：R中满足条件的元组，记作$\sigma_F(R)$，F为条件式
- 投影：R中若干属性组成新关系，记作$\Pi_A(R)$，A为R的属性列
- 连接：两个关系的笛卡尔积中选取属性满足条件$\theta$的元组，记作$R\bowtie_{A\theta B} S$
	- 等值连接：A=B
	- 自然连接：属性同名的等值连接
		- 外连接：保留含空值元组
			- 左外连接：保留左侧所有元组（左侧无空值）
			- 右外连接：保留右侧所有元组（右侧无空值）
- 除：R除S的结果包含在R但不在S的属性及值，记为$R\div S$

### 模式的定义与删除
- 定义：`CREATE SCHEMA [<模式名>] AUTHORIZATION <用户名>;`
- 删除：`DROP SCHEMA <模式名><CASCADE|RESTRICT>;`
	- CASCADE：级联，删除所有对象
	- RESTRICT：限制，没有对象时才能删除
### 基本表的定义、删除与修改
- 定义：`CREATE TABLE <表名>(<列名><数据结构>[列级完整性约束][,<列名><数据结构>[列级完整性约束]]...[,[表级完整性约束]]);`
- 数据类型：![[SQL数据类型.png]]
- 模式与表：
- 修改基本表：
	```
	ALTER TABLE <表名>
		[ADD[COLUMN]<新列名><数据类型>[完整性约束]]
		[ADD<表级完整性约束>]
		[DROP[COLUMN]<列名>[CASCADE|RESTRICT]]
		[RENAME COLUMN <列名> TO <新列名>]
		[ALTER COLUMN <列名> TYPE <数据类型>];
    ```
- 删除基本表：`DROP TABLE <表名>[CASCADE|RESTRICT];`
### 索引的建立与删除
- 索引可以加快查询速度
- 常见索引类型：顺序表索引、B+树索引、哈希索引、位图索引
- 建立索引：`CREATE[UNIQUE][CLUSTER]INDEX<索引名>`
- 修改索引：`ALTER INDEX <旧索引名>RENAME TO <新索引名>`
- 删除索引：`DROP INDEX <索引名>`
